## 一、设计语言

### 1.1 设计哲学

银河求索具身智能教育平台面向高职院校教学场景，以**桌面端机房使用**为主场景。界面设计遵循五项核心原则：

| 原则 | 释义 | 实践指引 |
|------|------|----------|
| **清晰 Clarity** | 内容优先，UI 为信息服务 | 通过视觉层级（字号/字重/色彩对比）与充裕留白引导注意力，而非依赖装饰 |
| **退让 Deference** | 界面退后，学习内容居前 | 控件在需要时出现，在不需要时隐去；视频/文本/实训环境占据最大视觉权重 |
| **纵深 Depth** | 投影与动效传递空间关系 | 卡片浮于底面之上，模态浮于页面之上；使用一致的投影阶梯（`shadow-sm/md/lg`）表达层级 |
| **一致 Consistency** | 相同操作 = 相同反馈 | 按钮行为、状态提示、交互模式在全平台保持统一；语义色彩令牌贯穿始终 |
| **可及 Accessibility** | WCAG 2.1 AA 为底线 | 键盘可操作、屏幕阅读器可读、色彩对比度达标、`prefers-reduced-motion` 受尊重 |

**美学方向**：科技教育感——理性、清晰、值得信赖。大面积留白搭配品牌蓝的克制点缀，用信息密度而非视觉装饰建立专业感。排斥过度渐变、过度圆角、emoji 图标等泛 AI 审美。

### 1.2 字体系统

采用系统字体栈，零延迟渲染，无 FOIT 风险：

```css
:root {
  --font-sans:
    -apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC",
    "Helvetica Neue", "Microsoft YaHei", sans-serif;
  --font-mono:
    "SF Mono", "Fira Code", "Cascadia Code", "Source Code Pro",
    "Menlo", "Consolas", monospace;
}
```

**字阶（Type Scale）**

基于 16px 基数，采用固定比例字阶，确保层级清晰：

| 语义名 | 字号 | 行高 | 字重 | 用途 | CSS 变量 |
|--------|------|------|------|------|----------|
| display | 32px | 40px | 700 | 登录页标题、数据看板大数字 | `--text-display` |
| heading-lg | 24px | 32px | 600 | 页面主标题 | `--text-heading-lg` |
| heading-md | 20px | 28px | 600 | 区域标题、模态标题 | `--text-heading-md` |
| heading-sm | 16px | 24px | 600 | 卡片标题、侧栏分组标题 | `--text-heading-sm` |
| body | 14px | 22px | 400 | 主体文本、表单标签 | `--text-body` |
| body-sm | 13px | 20px | 400 | 表格内容、紧凑列表 | `--text-body-sm` |
| caption | 12px | 18px | 400 | 次要说明、时间戳、状态标签 | `--text-caption` |

> **行宽控制**：正文容器限制 `max-width: 75ch`（约 60–75 字符/行）。长段落文本使用 `text-wrap: pretty` 优化断行。

**排版规范**

- 标题使用 `text-wrap: balance` 平衡换行
- 数据列、价格、计时器使用 `font-variant-numeric: tabular-nums` 等宽数字
- 省略号使用 `…`（U+2026），禁止 `...`
- 加载文案以 `…` 结尾（如"正在加载…"）
- 单位与数字之间使用不换行空格：`10&nbsp;MB`
- 截断策略：优先换行（wrap），必须截断时使用 `text-overflow: ellipsis` 并通过 Tooltip 展示完整内容
- Flex 子元素需设置 `min-w-0` 以支持文本截断

### 1.4 间距与栅格

基于 **8pt 栅格**，最小单位 4px。所有间距、尺寸取值限定为 4 的倍数。

| 令牌名 | 值 | 典型用途 | CSS 变量 |
|--------|-----|---------|----------|
| space-xs | 4px | 图标与文字间距、紧凑内边距 | `--space-xs` |
| space-sm | 8px | 表单项内间距、标签间距 | `--space-sm` |
| space-md | 16px | 卡片内边距、区域间分隔 | `--space-md` |
| space-lg | 24px | 区域级间距、侧栏与主区分隔 | `--space-lg` |
| space-xl | 32px | 页面级留白 | `--space-xl` |
| space-2xl | 48px | 大区块间距（登录页上下留白） | `--space-2xl` |

**布局约束**

- 内容区最大宽度：`1280px`，两侧自动居中
- 卡片网格 gutter：`24px`（`--space-lg`）
- 桌面端内容区水平 padding：`--space-xl`
- 优先使用 Flex / Grid 布局，避免 JS 测量计算布局

**圆角阶梯**

| 令牌名 | 值 | 用途 | CSS 变量 |
|--------|-----|------|----------|
| radius-sm | 6px | 按钮、输入框、标签 | `--radius-sm` |
| radius-md | 8px | 卡片、下拉菜单 | `--radius-md` |
| radius-lg | 12px | 模态弹窗 | `--radius-lg` |
| radius-full | 9999px | 头像、徽标 | `--radius-full` |

**投影阶梯（Elevation）**

| 令牌名 | 值 | 用途 | CSS 变量 |
|--------|-----|------|----------|
| shadow-sm | `0 1px 3px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04)` | 卡片静态 | `--shadow-sm` |
| shadow-md | `0 4px 12px rgba(0,0,0,0.08)` | 卡片悬浮、下拉面板 | `--shadow-md` |

### 1.6 动效系统

**时长与缓动**

| 属性 | 值 | 说明 | CSS 变量 |
|------|-----|------|----------|
| 微交互时长 | 150–200ms | 按钮态变、下拉展开、卡片出现 | `--duration-fast` |
| 退出时长 | 100–150ms | 收起、关闭（退出比进入快 ~70%） | `--duration-exit` |
| 复杂过渡时长 | 250–350ms | 页面切换、模态进入 | `--duration-normal` |
| 进入缓动 | `cubic-bezier(0.16, 1, 0.3, 1)` | 减速进入（ease-out 变体） | `--ease-out` |
| 退出缓动 | `cubic-bezier(0.4, 0, 1, 1)` | 加速退出 | `--ease-in` |
| 骨架屏脉冲 | 1.5s, `ease-in-out`, infinite | 加载占位 | — |

**动效规则**

- **仅动画 `transform` 和 `opacity`**——禁止动画 `width`/`height`/`top`/`left`，避免触发布局重排
- **禁止 `transition: all`**——必须显式列出属性名（如 `transition: transform 150ms var(--ease-out), opacity 150ms var(--ease-out)`）
- **设置正确的 `transform-origin`**——下拉从触发器展开、模态从触发源缩放
- **动画须可中断**——用户操作（点击、按键）须立即中断进行中的动画
- **尊重 `prefers-reduced-motion: reduce`**——跳过过渡动画，直接切换状态
- **列表/网格入场**：逐项交错 30–50ms（`animation-delay`），避免同时入场
- **模态/面板动效**：从触发源位置缩放+淡入，维持空间连续性

**按压反馈**

可交互卡片/按钮在 `:active` 时应用 `transform: scale(0.98)`，松开恢复，提供即时触感反馈。反馈须在 100ms 内可见。

### 1.7 图标规范

- 全部图标使用 **SVG 格式**，统一来自同一图标库（推荐 Lucide Icons 或 Heroicons）
- 基准尺寸 20×20px（导航图标 24×24px），通过设计令牌 `--icon-sm: 16px`、`--icon-md: 20px`、`--icon-lg: 24px` 管理
- 保持统一的描边宽度（1.5px 或 2px），禁止在同一层级混用 filled 和 outline 风格
- **禁止将 emoji 用作结构性图标**（导航、设置、系统控件等）
- 纯装饰性图标设置 `aria-hidden="true"`
- 可交互图标按钮**必须**提供 `aria-label`
- 图标与文字对齐到基线，保持一致的 padding

## 二、组件系统

### 2.1 按钮系统

全平台按钮分四级，**每屏至多一个主按钮**：

| 类型 | 样式 | 规则 |
|------|------|------|
| 主按钮 Primary | `brand-primary` 底 + 白色文字 | **每屏至多一个**。用于当前上下文最核心的行动 |
| 次按钮 Secondary | 白底 + `brand-primary` 边框与文字 | 次要操作，如"取消"、"重置筛选" |
| 文字按钮 Text | 无底无边框，`brand-primary` 文字 | 行内操作，如"查看详情"、"展开" |
| 危险按钮 Danger | `error` 底 + 白色文字 | 不可逆操作，如"删除"、"停用"。须与常规按钮保持视觉与空间分隔 |

**按钮状态矩阵**

| 状态 | 表现 | 技术要求 |
|------|------|----------|
| Default | 默认样式 | — |
| Hover | 背景加深（`brand-primary-hover`） | `cursor: pointer`；hover 态须增加对比度 |
| Active | `transform: scale(0.98)` | 100ms 内可见反馈 |
| Focus | 2px `brand-primary` 外发光 | 仅 `:focus-visible` 可见，鼠标点击不显示 |
| Disabled | 40% 不透明度 | `cursor: not-allowed`；移除所有 hover 效果；设置 `disabled` 属性（非仅视觉） |
| Loading | 文字替换为 spinner + "处理中…" | 禁止重复点击；提交开始后才禁用按钮，提交前保持可用 |

- 按钮最小高度 36px，内边距 `12px 20px`，文字 14px/600
- 使用 `<button>` 标签处理动作，使用 `<a>`/`<Link>` 处理导航——**禁止 `<div onClick>` 模拟按钮**
- 按钮文案使用具体动词（"提交任务"而非"确定"；"导入学生"而非"确认"）

### 2.2 表单系统

**标签**
- 始终可见于输入框上方（`body` 字号 + `text-primary` 颜色）
- 必填项标签后追加红色 `*`
- **禁止仅用 placeholder 充当标签**
- `<label>` 与 `<input>` 通过 `for`/`id` 关联，确保 label 可点击聚焦

**输入框**
- 高度 36px，`border-default` 颜色 1px 边框，`radius-sm` 圆角
- 聚焦时边框变为 `border-focus`，附加 `0 0 0 2px var(--color-brand-primary-light)` 外发光
- Placeholder 以 `…` 结尾，展示示例格式（如 `student@example.edu…`）
- 使用正确的 `type` 属性（`email`、`tel`、`url`、`number`）触发对应移动键盘
- 设置 `autocomplete` 和有意义的 `name` 属性
- 邮箱、用户名、验证码字段设置 `spellCheck={false}`
- **禁止 `onPaste + preventDefault` 阻止粘贴**

**校验**
- **失焦时校验**（validate on blur），非逐键校验——避免用户输入过程中被打断
- 提交时对所有字段再校验一次
- 校验失败后**自动聚焦到第一个错误字段**
- 错误提示通过 `aria-describedby` 关联至输入框，使用 `aria-live` region 或 `role="alert"` 通知屏幕阅读器

**错误提示**
- 显示于字段下方，`caption` 字号 + `error` 颜色，附带错误图标
- 输入框边框同步变为 `error`
- 错误信息**须说明原因和修复方式**（如"邮箱格式不正确，请输入如 name@school.edu 的有效邮箱"——而非仅"输入无效"）
- 多字段错误时，顶部展示错误摘要，附锚点链接跳转至对应字段

**其他控件**
- **下拉选择**：与输入框等高，右侧箭头图标 SVG 渲染。原生 `<select>` 须显式设置 `background-color` 和 `color`（深色模式兼容）
- **文件上传**：虚线边框拖拽区，支持点击选择。上传中显示进度条，上传完成显示文件名 + 删除图标
- **密码输入**：提供明文切换按钮（眼睛图标 + `aria-label`）
- **复选框/单选框**：label 和控件共享同一点击热区
- **禁用态**：使用降低不透明度 0.38–0.5 + `cursor` 变更 + 语义 `disabled` 属性（非仅视觉）

**长表单保护**
- 含未保存修改的表单/模态，离开前须确认提示（"您有未保存的更改，确定离开吗？"）
- 长表单（如院校新建/编辑）应自动保存草稿至 localStorage，防止意外丢失

### 2.3 状态反馈

全平台使用统一的状态反馈组件：

**加载态 Loading**

| 场景 | 实现方式 |
|------|----------|
| 页面级 | 骨架屏（Skeleton）——灰色矩形块模拟内容布局，1.5s 脉冲动画 |
| 局部刷新 | 区域内居中 spinner（`brand-primary`，24px） |
| 按钮提交 | 按钮进入 Loading 状态（spinner + "处理中…"） |
| 超过 300ms 的操作 | 必须展示加载指示器，不允许无反馈等待 |

**空态 Empty**
- 居中插画（SVG，120×120px）+ 一行灰色说明文字 + 可执行操作按钮（如有）
- 文案具体化：不说"暂无数据"，而说"暂无课程，请联系管理员分配"

**错误态 Error**

| 场景 | 实现方式 |
|------|----------|
| 网络错误 | 错误插画 + "加载失败" + **重试按钮**（须提供恢复路径） |
| 表单错误 | 见 §2.2 表单系统 |
| 操作失败 | 顶部滑入 Toast（`error` 背景 + 白色文字），3–5s 自动消失，支持手动关闭 |
| 请求超时 | 明确提示"请求超时" + 重试选项 |

**成功态 Success**

| 场景 | 实现方式 |
|------|----------|
| 普通操作 | 顶部滑入 Toast（`success` 背景 + 白色文字），2s 自动消失 |
| 关键成功（如提交实训） | 轻量 Success Modal |
| 危险操作完成 | Toast 提示操作结果（如"已禁用账号 张三"） |

**Toast 规范**
- 自动消失时长：成功 2s，错误 3–5s
- 使用 `aria-live="polite"` 通知屏幕阅读器——**不抢夺焦点**
- 对比度须满足 4.5:1

### 2.4 模态与弹窗

**模态弹窗 Modal**
- `radius-lg` 圆角，`shadow-lg` 投影
- 半透明遮罩 `var(--color-bg-overlay)` / `rgba(0,0,0,0.4)`——足以隔离背景内容
- 尺寸：480px（小型）/ 640px（中型）/ 800px（大型）
- 顶部标题 + 右上角关闭按钮，底部操作按钮右对齐
- 点击遮罩或按 `Esc` 关闭
- 动效：从触发源位置 `scale(0.95)` + `opacity: 0` → `scale(1)` + `opacity: 1`，250ms `ease-out`；关闭 150ms
- 遮罩设置 `overscroll-behavior: contain` 防止滚动穿透

**模态交互规则**
- 打开时：`body` 禁止滚动，焦点困在模态内（focus trap），`inert` 背景内容
- 关闭后：焦点回到触发元素
- 含未保存修改的模态关闭前需确认提示
- **模态不用于主导航流程**——模态是中断，不是路径

**确认弹窗 Confirm**
- 模态子类型，固定 480px 宽
- 危险操作使用危险按钮，且危险按钮与常规按钮保持视觉分隔
- 确认文案须具体（"确定禁用学生 张三 的账号吗？禁用后无法登录，数据保留。"——而非"确定吗？"）。**注意：本平台不支持永久删除账号，仅通过禁用处理（PRD §5.1.2）**

**抽屉 Drawer**
- v2.0 考虑用于移动端适配。v1.0 不使用

### 2.5 表格组件

**基本规范**
- 表头背景 `bg-page`，正文交替行背景或统一白底
- 行高 ≥48px，确保文字可读和交互热区
- 当前排序列通过 `aria-sort` 标注，支持键盘操作
- 响应式：窄屏下允许水平滚动（`overflow-x: auto`），并给出滚动指示阴影（左右渐变遮罩）
- ≥50 行使用虚拟滚动（react-window 或同类方案）

**操作列**
- 操作按钮使用文字按钮样式
- 操作 ≥3 项时收入"更多"溢出菜单（`overflow menu`），避免拥挤

  - "退出登录"

### 3.3 路由结构与权限守卫

```
/login                           ← 唯一公开页面
/change-password                 ← 所有登录角色
/profile                         ← 个人信息编辑（教师/院校管理员/企业管理员，学生无权限）

# 学生 + 教师共享
/courses                         ← 课程列表
/courses/:courseId                ← 课程详情（含课时学习）

# 学生 + 教师共享
/training                        ← 实训包列表
/training/:packId                ← 实训任务列表（含"我的提交"Tab）
/training/:packId/tasks/:taskId  ← 实训任务
/training/submissions            ← 我的提交记录

# 教师独享
/training/review                 ← 批改列表
/training/submissions/:id/review ← 批改详情

# 院校管理员独享
/accounts                        ← 账号管理
/courses/assign                  ← 课程分配
/training/assign                 ← 实训分配
/courses/browse                  ← 课程内容浏览
/training/browse                 ← 实训内容浏览（查看任务说明与参考答案）

# 企业管理员独享
/schools                         ← 院校列表
/schools/:id                     ← 院校详情
/schools/new                     ← 新建院校
/schools/:schoolId/edit          ← 编辑院校
/content                         ← 内容管理落地页（重定向至 /content/courses）
/content/courses                 ← 课程内容管理
/content/courses/:id/edit        ← 课程编辑
/content/training                ← 实训内容管理
/content/training/:id/edit       ← 实训编辑
```

**路由守卫逻辑**

1. 未登录用户访问任何非 `/login` 路由 → 重定向至 `/login`
2. 已登录用户访问 `/login` → 重定向至该角色落地路由
3. 已登录用户访问无权限路由 → 显示 403 页面（含"返回首页"按钮）
4. **URL 始终反映当前状态**——筛选条件、分页参数、Tab 状态通过 query string 持久化，支持浏览器前进/后退和链接分享
5. 页面跳转使用 `<a>` / `<Link>` 标签（支持 Cmd/Ctrl+click 新标签页），**禁止 `<div onClick>` + `router.push` 模拟导航**
6. 返回导航须可预期且一致——保留滚动位置、筛选状态、输入内容

**路由切换后焦点管理**
- 页面切换后，焦点移至主内容区域（`<main>`），确保屏幕阅读器用户知道页面已更新

### 3.4 响应式策略

本平台以桌面端（高职院校机房）为主场景，采用 mobile-first 向上适配：

| 断点 | 宽度范围 | 策略 |
|------|---------|------|
| Mobile | < 768px | 卡片单列，登录页隐藏左面板，侧栏变为可折叠 |
| Tablet | 768–1023px | 卡片网格降为 2 列，侧栏收窄 |
| Desktop | ≥ 1024px | 标准布局 |

**技术要求**

- `<meta name="viewport" content="width=device-width, initial-scale=1">`——**禁止 `user-scalable=no` 或 `maximum-scale=1` 禁用缩放**
- 禁止出现水平滚动条（表格除外，见 §2.5）
- 图片使用 `max-width: 100%` 自适应
- 使用 `min-h-dvh` 替代 `100vh`（移动端视口兼容）
- 大屏（≥1440px）内容区水平 gutter 适当加宽
- 横屏方向须保持布局可读和可操作

```
│  任务描述(折叠)/          │   快捷分数 [60-100]   │
│  参考答案(折叠)           │   评语文本框          │
│                         │   [评分] / [打回重做]  │
├─────────────────────────┴──────────────────────┤
│  [上一份] [下一份]           [完成批改] 底部栏    │
└────────────────────────────────────────────────┘
```

- 评分面板 `position: sticky`，`top` 值须考虑固定导航栏高度
- 成绩 0–100 整数，快捷按钮 60/70/80/90/100
- 批改操作二选一：**评分**（给分 + 反馈）或 **打回重做**（仅反馈，无分数）
- 已批改过时顶部橙色提示"历史评分：N 分，当前操作将覆盖"
- 底部切换栏："← 上一份" / "第 N/M 条" / "下一份 →"（快速切换无需返回列表）
- 历史提交与对应批改记录可展开查看

表格列：院校名称、地区、学生数（已/上限）、教师数（已/上限）、授权到期日、状态（正常/已过期/已停用）、操作。

**新建/编辑院校**：纵向多卡片区块——

5. **总授权有效期**
6. **院校管理员**：账号信息

**院校详情** `/schools/:id`：只读信息卡片 + Tab（教师列表/学生列表）。

- 追加/移除课程或实训包授权
- 调整单项配额和有效期
- 延长或缩短总授权有效期

#### 4.5.2 课程内容管理 `/content/courses`

导航标签"内容管理"为一级入口，子导航通过 Tab 切换"课程管理"（`/content/courses`）和"实训管理"（`/content/training`），两者为独立路由。

**课程列表表格**列：课程名称、岗位方向、技能等级、使用院校数、上架状态（开关切换 + 确认弹窗）、更新时间、操作。

**上架/下架语义**（PRD §5.1.3）：
- **下架** = 内容管理操作，内容不再对外可见，已分配学生**不受影响**可继续学习/提交
- **授权到期/取消** = 院校授权操作，已分配学生**立即冻结**——课程 v1.0 从列表消失，实训包标注"已过期/已取消"且无法新提交

**为教师开启内容编辑权限**（PRD §5.2.5）：院校详情页 → 教师列表 → 每位教师行的"更多"菜单中提供"开启/关闭内容编辑权限"开关。开启后该教师可编辑其绑定的课程/实训包内容。

**课程编辑** `/content/courses/:id/edit`：左右双栏（35:65）。

左侧基本信息（名称、描述、封面图、岗位方向、技能等级、学时）。

右侧章节编辑（可拖拽排序的树形结构编辑器）：

```
第一章 [章节名称]                          [拖拽柄] [删除]
  ├─ 1.1 [课时名称] [类型: 视频▼]         [拖拽柄] [删除]
  │       └─ [上传/选择视频文件]
  ├─ 1.2 [课时名称] [类型: 富文本▼]        [拖拽柄] [删除]
  │       └─ [富文本编辑器]
  └─ + 添加课时

+ 添加章节
```

- 拖拽排序：拖拽过程中禁用文字选择（`user-select: none`），被拖拽元素使用 `inert`
- 发布前校验：课程名称必填、至少一个章节、每章至少一个课时、每课时内容不为空
- 校验失败后聚焦到第一个错误项

#### 4.5.3 实训内容管理 `/content/training`

实训编辑：基本信息 + 可排序任务列表（任务名称 + 描述富文本编辑器 + 参考答案 + 云桌面类型下拉）。

## 五、边界与异常处理

> 本节集中定义全平台的边界场景处理逻辑，与 PRD §5.1.3、§5.2.6、§5.3.6 严格对齐。所有前端展示须同时在后端 API 层做校验。

### 5.1 院校级异常

| 场景 | 前端处理 | 后端处理 |
|------|----------|----------|
| 院校总授权到期 | 登录页提示"授权已到期，请联系管理员"；已登录用户强制登出 | 登录接口返回特定错误码 |
| 院校已停用 | 登录页提示"院校已停用，请联系管理员"；已登录用户强制登出 | 登录接口返回特定错误码 |
| 院校管理员已创建但未配置任何授权 | 院校管理员可正常登录和管理账号；课程/实训包分配页显示空列表 + 提示"暂无可分配的课程/实训包，请联系银河管理员" | 正常返回空列表 |

### 5.2 账号级异常

| 场景 | 前端处理 | 后端处理 |
|------|----------|----------|
| 账号已禁用 | 登录页提示"账号已禁用，请联系您的管理员" | 登录接口返回特定错误码 |
| 学生账号数达上限 | "手动新增"按钮 Disabled + Tooltip"已达学生账号上限（N/N），请联系银河管理员" | 创建接口返回 422 + 错误信息 |
| 教师账号数达上限 | 同上 | 同上 |
| 批量导入超过剩余配额 | 整批拒绝，提示"导入人数（N）超过剩余配额（M），请减少导入数量或联系银河管理员扩容" | 导入接口返回 422 + 错误信息 |
| 禁用学生不释放配额 | 禁用确认弹窗中明确说明"账号仍占用配额。如需释放配额，请先取消课程/实训包分配" | 禁用不自动取消分配 |

### 5.3 课程授权异常

| 场景 | 前端处理 |
|------|----------|
| 单门课程授权到期 | 该课程对新学生不可分配（分配页不显示）；已分配学生的课程**立即冻结**——v1.0 课程从学生列表中消失，学习记录后台保留 |
| 课程使用配额已满 | 分配时阻止，提示"该课程已达分配上限（已分配 N / 配额 N），请联系银河管理员扩容" |
| 企业管理员缩减课程授权（移除） | 处理方式与课程授权到期一致：已分配学生课程立即冻结，v1.0 从列表消失 |
| 企业管理员降低配额（低于已分配数） | 不影响已分配学生，但院校管理员无法继续分配新学生 |
| 课程被下架 | 已分配学生**不受影响**，可继续学习；新分配入口对院校管理员不可见 |

### 5.4 实训包授权异常

| 场景 | 前端处理 |
|------|----------|
| 单个实训包授权到期 | 该实训包对新学生不可分配；已分配学生可查看历史提交和批改记录，但**无法发起新提交和打开实训环境**；列表中标注"已过期" |
| 实训包使用配额已满 | 分配时阻止，提示"该实训包已达分配上限（已分配 N / 配额 N），请联系银河管理员扩容" |
| 企业管理员缩减实训包授权（移除） | 处理方式与到期一致；列表中标注"授权已取消" |
| 实训包被下架 | 已分配学生**不受影响**，可继续提交和查看历史记录；新分配入口不可见 |

### 5.5 教师绑定异常

| 场景 | 前端处理 |
|------|----------|
| 教师被取消课程绑定 | 该课程立即从教师视图消失，教师无法再查看该课程的学生进度；已产生的进度数据不受影响 |
| 教师被取消实训包绑定 | 该实训包立即从教师视图消失，教师无法再查看和批改该实训包下的提交；已完成的批改记录不受影响 |
| 学生班级变更 | 原班教师**立即不可见**该学生（含进度和提交记录），新班教师**立即可见**该学生全部数据；已完成的批改记录不受影响 |

### 5.6 学习与实训异常

| 场景 | 前端处理 |
|------|----------|
| 学生尝试访问未解锁小节 | 点击无响应，Tooltip"请先完成前置课时" |
| 实训环境连接异常 | 展示重试入口 + 错误信息，错误写入后台日志 |
| 学生多次提交同一任务 | 每次独立记录（含对应批改），教师批改最新一次，历史记录均可查 |
| 教师重复批改同一提交 | 允许覆盖，更新分数/状态和反馈 |
| 学生提交后想撤回 | 不支持撤回；可直接重新提交，新提交自动成为最新记录 |
| 教师打回重做后学生未重新提交 | 最新提交状态停留在"打回重做"，不影响其他任务 |

## 六、数据模型

> 本节集中定义全平台核心数据模型，供后端和前端开发参考。

### 6.1 院校对象（School）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 院校名称 |
| region | string | 所在地区 |
| student_limit | int | 学生账号上限 |
| teacher_limit | int | 教师账号上限 |
| expires_at | timestamp | 院校总授权到期时间 |
| status | enum | active / expired / deactivated |

> **三态说明（PRD §4.4.4）**：`active` = 正常，所有用户可登录；`expired` = 总授权到期，所有用户无法登录；`deactivated` = 手动停用，所有用户无法登录。`expired` 和 `deactivated` 行为一致但恢复路径不同（续期 vs 恢复）。

### 6.2 院校课程授权（SchoolCourseAuthorization）

> PRD §5.1.1：每门课程单独设置学生使用配额和授权有效期。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| school_id | UUID | 院校 |
| course_id | UUID | 课程 |
| student_quota | int | 该课程可分配的学生上限 |
| expires_at | timestamp | 该课程的授权有效期 |

### 6.3 院校实训包授权（SchoolPackAuthorization）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| school_id | UUID | 院校 |
| pack_id | UUID | 实训包 |
| student_quota | int | 该实训包可分配的学生上限 |
| expires_at | timestamp | 该实训包的授权有效期 |

### 6.4 用户对象（User）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| school_id | UUID | 所属院校（企业管理员为空） |
| role | enum | student / teacher / school_admin / platform_admin |
| name | string | 姓名 |
| email | string | 邮箱（登录用，院校内唯一——复合唯一约束 `school_id + email`） |
| phone | string | 手机号 |
| class_name | string | 班级（学生） |
| student_no | string | 学号（学生，选填） |
| force_change_password | bool | 是否强制改密 |
| is_active | bool | 是否启用 |
| content_edit_enabled | bool | 是否开启内容编辑权限（教师，默认 false，企业管理员控制） |

### 6.5 教师绑定关系（TeacherBinding）

> PRD §4.3：教师数据范围 = 已绑定课程/实训包 ∩ 已绑定班级的学生。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| teacher_id | UUID | 教师 |
| course_id | UUID | 绑定课程（可为空） |
| pack_id | UUID | 绑定实训包（可为空） |
| class_name | string | 绑定班级 |

> 教师与课程/实训包/班级为多对多关系。每条记录表示一个绑定维度。绑定关系可由院校管理员在创建教师时设置，也可随时修改。

### 6.6 学生课程分配记录（StudentCourseAssignment）

> PRD §5.2.4：分配计入课程使用配额，取消分配释放配额。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| student_id | UUID | 学生 |
| course_id | UUID | 课程 |
| school_id | UUID | 院校（冗余，便于配额计数） |
| assigned_at | timestamp | 分配时间 |

### 6.7 学生实训包分配记录（StudentPackAssignment）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |

## 七、技术规范

### 7.1 安全要求

| 领域 | 要求 |
|------|------|
| 认证 | JWT Token 存储于 httpOnly cookie，**禁止 localStorage 存储 Token** |
| 会话 | Token 有效期 ≤8 小时，过期后重定向至登录页 |
| 权限 | 前端路由守卫 + 后端 API 双重校验。前端隐藏无权限入口，后端拒绝越权请求返回 403 |
| XSS | 富文本内容使用白名单 HTML sanitizer 渲染，**禁止 `dangerouslySetInnerHTML` 裸用** |
| CSRF | API 请求携带 CSRF Token |
| 文件上传 | 前端校验文件类型与大小，后端二次校验。允许类型：jpg/png/gif/webp/pdf/docx/xlsx/zip/mp4/webm。单文件 ≤50MB |
| 密码 | 前端不明文展示密码（重置密码弹窗除外），传输使用 HTTPS |
| 敏感操作 | 删除、停用等不可逆操作必须二次确认弹窗，确认文案须具体 |
| 登录错误 | 不透露"邮箱不存在"或"密码错误"——统一返回"邮箱或密码错误" |

### 7.2 可及性清单（WCAG 2.1 AA）

**感知**
- [ ] 色彩对比度：正文 ≥4.5:1，大标题 ≥3:1
- [ ] 图片替代文本：有意义的 `alt` 文本；纯装饰 `alt=""`
- [ ] 色彩非唯一信号：不单独依赖颜色传达信息，同时使用图标或文字
- [ ] 减弱动效：`prefers-reduced-motion: reduce` 时禁用过渡动画

**操作**
- [ ] 键盘导航：所有可交互元素可通过 Tab/Shift+Tab 到达，Enter/Space 激活
- [ ] 焦点指示：2px `brand-primary` outline，offset 2px。鼠标用户不显示（`:focus-visible`）。**禁止 `outline: none` 不提供替代**
- [ ] 跳过链接：页面顶部隐藏的"跳转到主要内容"链接
- [ ] 图标按钮标注：无文字图标按钮必须有 `aria-label`
- [ ] 键盘快捷键：保留系统和辅助功能快捷键；拖拽排序提供键盘替代方案

**理解**
- [ ] 语义标签：使用 `<nav>`、`<main>`、`<aside>`、`<header>`、`<footer>`、`<section>`
- [ ] 标题层级：h1→h6 顺序递进，不跳级
- [ ] 表单关联：`<label>` 与 `<input>` 通过 `for`/`id` 关联。错误提示通过 `aria-describedby` 关联
- [ ] 导航使用 `<a>`：页面跳转使用 `<a href>` 而非 `<div onClick>`
- [ ] 错误提示：说明原因和修复方式，使用 `aria-live` 通知

**健壮**
- [ ] 动态内容通知：Toast 使用 `aria-live="polite"`
- [ ] 焦点管理：模态 focus trap；关闭后焦点回到触发元素；路由切换后焦点移至 `<main>`
- [ ] 输入水合安全：带 `value` 的输入须有 `onChange`（或用 `defaultValue`）；日期/时间渲染注意水合不匹配

### 7.3 性能要求

| 指标 | 目标值 | 实现手段 |
|------|--------|---------|
| 首屏加载 (LCP) | ≤2.5s（校园网环境） | 路由级代码分割、关键 CSS 内联、`<link rel="preconnect">` CDN 域 |
| 交互延迟 (INP) | ≤200ms | 避免主线程长任务（每帧 ≤16ms）；高频事件防抖/节流 |
| 累计布局偏移 (CLS) | ≤0.1 | 图片/视频 `width`+`height` 预设、骨架屏占位、`font-display: swap` |
| JS 打包体积 | 首屏 ≤200KB (gzip) | Tree shaking、动态 import、按需加载 |
| 图片加载 | WebP/AVIF 优先 | 首屏外 `loading="lazy"`；首屏关键图 `fetchpriority="high"` |
| 长列表 | ≥50 条使用虚拟滚动 | react-window / 类似方案 |
| 高频事件 | 防抖/节流处理 | 搜索输入防抖 300ms、滚动节流 |
| 骨架屏 | 所有异步内容区域 | 页面级 + 卡片级 |
| 缓存策略 | 静态资源 content hash + 长期缓存 | Webpack/Vite content hash |
| 字体 | 关键字体预加载 | `<link rel="preload" as="font">`，`font-display: swap` |
| 第三方脚本 | async/defer 加载 | 审计并移除不必要的第三方脚本 |
| 表单输入 | 优先使用非受控输入 | 减少不必要的 re-render |
| 布局计算 | 批量 DOM 读写 | 避免在渲染中读取布局属性 |

### 7.4 浏览器兼容性

| 浏览器 | 最低版本 |
|--------|---------|
| Chrome | 90+ |
| Firefox | 90+ |
| Edge | 90+ |
| Safari | 14+ |

不支持 IE。不支持的浏览器显示提示横幅"请使用 Chrome/Edge/Firefox 最新版本访问本平台"。

### 7.5 国际化与格式化预留

- 文案使用 key-value 管理，不硬编码中文字符串
- 日期/时间使用 `Intl.DateTimeFormat` 格式化
- 数字/货币使用 `Intl.NumberFormat` 格式化
- 时间格式统一 ISO 8601（`2026-05-15T10:30:00Z`）

## 九、开发前 Checklist

在每个功能模块开发完成后，对照以下清单进行自检：

### 视觉质量
- [ ] 未使用 emoji 作为结构性图标（使用 SVG 图标库）
- [ ] 图标来自同一图标库，风格统一
- [ ] 语义色彩令牌贯穿使用，无硬编码 hex 色值
- [ ] 投影层级使用设计令牌阶梯，无随机 `box-shadow`
- [ ] 间距遵循 4/8pt 栅格节奏

### 交互
- [ ] 所有按钮/链接有 hover 态，且 hover 态增加对比度
- [ ] 危险操作有确认弹窗，确认文案具体
- [ ] 操作成功/失败均有明确反馈（Toast / Modal / 内联提示）
- [ ] Loading 超过 300ms 有加载指示器
- [ ] Disabled 元素有语义属性（不仅视觉灰化）

- [ ] 每个输入有可见 label（非仅 placeholder）

- [ ] 失焦校验 + 提交全量校验
- [ ] 错误提示在字段下方，说明原因和修复方式
- [ ] 提交失败后自动聚焦第一个错误字段

### 可及性
- [ ] 色彩对比度达标（正文 ≥4.5:1）
- [ ] 键盘可完成所有操作
- [ ] 焦点指示可见（`:focus-visible`）
